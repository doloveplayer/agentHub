# Phase 3.5 Task 0 — REPL Persistent Agent Architecture Upgrade

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development or superpowers:executing-plans.
> **Parent Spec:** `docs/superpowers/specs/2026-05-21-repl-persistent-agent.md`
> **Priority:** P0 — MUST complete before Phase 3.5 Tasks 1-5

**Goal:** Replace one-shot `docker exec claude --print` with `child_process.spawn('docker', ['run', '-i', ...])` native pipes. Each agent gets an independent container with isolated CLAUDE_CONFIG_DIR (memory, skills, CLAUDE.md) while sharing the workspace filesystem.

**Architecture:** `spawn('docker', ['run', '-i', ...])` creates per-agent containers with bind-mounted workspace + agent config. Native Node.js stdin/stdout pipes replace Docker multiplex protocol entirely. `--rm` auto-cleans containers on exit. Handler lazily initializes agent directories on first @mention.

**Tech Stack:** child_process.spawn, docker run -i, native pipes (no multiplex), Claude Code --print mode

**Dependencies:** None — this is the new foundation for Phase 3.5

**Status (2026-05-21):** Core implementation complete. Tasks 0-1 through 0-6, 0-8 done. Task 0-7 (REPL process reuse) deferred to Phase 3.5 Task 1.

## Why This MUST Be First

All Phase 3.5 features depend on persistent agent processes:

| Phase 3.5 Feature | Why It Needs REPL |
|-------------------|-------------------|
| InboxManager | Needs stdin access to inject inbox-check reminders between tool_use cycles |
| MilestoneBroadcaster | Needs persistent process to detect phase boundaries in real-time |
| Sequential mode (A→B) | Needs Agent A's process alive to pass context to Agent B |
| Intervention flow | Needs stdin to deliver intervention_request to target agent |
| PTY Permission (#10) | REPL mode emits `permission_request` events that `--print` suppresses |

Building Phase 3.5 on `--print` mode would require a complete rewrite later.

// apps/api/src/agent/providers/base.ts
export interface UnifiedAgentEvent {
  type: 'thinking' | 'tool_use' | 'tool_result' | 'subagent_start'
      | 'subagent_result' | 'permission_request' | 'milestone'
      | 'blocked' | 'done' | 'error';
  content?: string;
  toolName?: string;
  toolInput?: Record<string, unknown>;
  filePath?: string;
  exitCode?: number;
  message?: string;
  providerRaw?: unknown;
  timestamp: number;
}

export interface ProviderConfig {
  apiKey?: string;
  baseUrl?: string;
  model?: string;
  env?: Record<string, string>;
  agentName?: string;
  hostWorkDir?: string;
}

export type EventHandler = (event: UnifiedAgentEvent) => void;

export interface AbstractProvider {
  readonly name: string;
  readonly capabilities: {
    persistentSession: boolean;
    permissionProxy: boolean;
    streamingOutput: boolean;
    independentMemory: boolean;
    independentConfig: boolean;
  };

  /** Start or resume a REPL session. Returns immediately, events via onEvent. */
  start(
    sessionId: string,
    prompt: string,
    containerId: string,
    workDir: string,
    config: ProviderConfig,
  ): Promise<void>;

  /** Send a new prompt to an existing REPL process (no new spawn) */
  sendPrompt(prompt: string): void;

  /** Write raw input to process stdin (permission responses, control) */
  write(input: string): void;

  /** Kill the REPL process */
  stop(): void;

  /** Register event handler */
  onEvent(handler: EventHandler): void;

  /** Check if the REPL process is currently alive */
  isAlive(): boolean;

  /** Get the agent's home directory inside the container */
  getAgentHome(): string;
}

import { mkdirSync, writeFileSync, existsSync } from 'fs';
import { resolve } from 'path';

export class AgentDirectoryManager {
  /**
   * Initialize per-agent directory structure inside the sandbox.
   *
   * Structure:
   *   {hostWorkDir}/_agent_{agentName}/
   *     CLAUDE.md
   *     .claude/
   *       memory/         → bind-mounted or symlinked
   *       skills/         → empty dir for user-provided skills
   */
  static initialize(
    hostWorkDir: string,
    agentName: string,
    systemPrompt: string,
  ): string {
    const agentDir = resolve(hostWorkDir, `_agent_${agentName}`);
    if (!existsSync(agentDir)) {
      mkdirSync(agentDir, { recursive: true });
      mkdirSync(resolve(agentDir, '.claude', 'memory'), { recursive: true });
      mkdirSync(resolve(agentDir, '.claude', 'skills'), { recursive: true });
    }

    // Write CLAUDE.md with agent's system prompt + collaboration rules
    const claudeMd = `# Agent: ${agentName}

${systemPrompt}

## Collaboration Rules
- You are part of a multi-agent session. Other agents may observe your work.
- Your workspace is at /workspace. All agents share this filesystem.
- Your personal files (memory, config) are at /workspace/_agent_${agentName}/
- When you complete a significant phase, note it in your output.
- You may be contacted by other agents. Check your inbox at /workspace/_agent_${agentName}/_inbox.jsonl
`;

    writeFileSync(resolve(agentDir, 'CLAUDE.md'), claudeMd, 'utf-8');

    return agentDir;
  }

  /** Clean up agent directory on session close */
  static cleanup(hostWorkDir: string, agentName: string): void {
    const agentDir = resolve(hostWorkDir, `_agent_${agentName}`);
    // Keep memory dir for now; full cleanup on session destroy
  }
}

### Step 1: REPL Shell Command

The key change: remove `--print`, use `cat file - | claude` so stdin stays open for subsequent prompts.

```typescript
// apps/api/src/agent/providers/claude-code.ts
import { AbstractProvider, EventHandler, ProviderConfig, UnifiedAgentEvent } from './base.js';
import { SandboxManager } from '../SandboxManager.js';
import { EventParser } from '../EventParser.js';
import { writeFileSync } from 'fs';
import { resolve } from 'path';
import { buildSafeEnv } from '../ClaudeCodeProcess.js';

export class ClaudeCodeProvider implements AbstractProvider {
  readonly name = 'claude-code';
  readonly capabilities = {
    persistentSession: true,
    permissionProxy: true,
    streamingOutput: true,
    independentMemory: true,
    independentConfig: true,
  };

  private containerId: string | null = null;
  private handlers: EventHandler[] = [];
  private killed = false;
  private partialLine = '';
  private stdinStream: NodeJS.WritableStream | null = null;
  private agentHome = '/workspace';

  onEvent(handler: EventHandler): void {
    this.handlers.push(handler);
  }

  private emit(event: UnifiedAgentEvent): void {
    for (const h of this.handlers) {
      try { h(event); } catch { /* isolate */ }
    }
  }

  getAgentHome(): string {
    return this.agentHome;
  }

  isAlive(): boolean {
    return !this.killed && this.stdinStream !== null;
  }

  async start(
    sessionId: string,
    prompt: string,
    containerId: string,
    workDir: string,
    config: ProviderConfig,
  ): Promise<void> {
    this.killed = false;
    this.containerId = containerId;
    this.agentHome = `/workspace/_agent_${config.agentName || 'agent'}`;

    // REPL mode: no --print. Claude Code stays alive waiting for next stdin line.
    const args = ['--output-format', 'stream-json', '--verbose'];

    // In REPL mode with persistentSession, we DON'T skip permissions.
    // Claude Code will emit permission_request events which we route to frontend.
    // DO NOT pass --dangerously-skip-permissions

    const safeEnv = buildSafeEnv();
    if (config.apiKey) safeEnv['ANTHROPIC_API_KEY'] = config.apiKey;
    if (config.baseUrl) safeEnv['ANTHROPIC_BASE_URL'] = config.baseUrl;

    // Set CLAUDE_CONFIG_DIR to agent's personal directory
    const agentConfigDir = `${this.agentHome}/.claude`;
    const envFile = config.hostWorkDir
      ? resolve(config.hostWorkDir, '_env.sh')
      : '/workspace/_env.sh';

    // Write env vars
    if (config.hostWorkDir) {
      const authKeys = ['ANTHROPIC_API_KEY', 'ANTHROPIC_BASE_URL', 'ANTHROPIC_AUTH_TOKEN'];
      const envLines: string[] = [];
      envLines.push(`export CLAUDE_CONFIG_DIR='${agentConfigDir}'`);
      for (const k of authKeys) {
        if (safeEnv[k]) {
          envLines.push(`export ${k}='${String(safeEnv[k]).replace(/'/g, "'\\''")}'`);
        }
      }
      writeFileSync(resolve(config.hostWorkDir, '_env.sh'), envLines.join('\n'), 'utf-8');
    }

    // Write initial prompt to file for stdin delivery
    const promptFile = `_repl_prompt_${config.agentName}.txt`;
    if (config.hostWorkDir) {
      writeFileSync(resolve(config.hostWorkDir, promptFile), prompt + '\n', 'utf-8');
    }

    // REPL command: cat file - | claude
    // - cat outputs the file, then `-` reads from Docker exec stdin
    // - stdin stays open for sendPrompt() and write()
    const shellCmd = `. /workspace/_env.sh && cd ${workDir} && cat /workspace/${promptFile} - | claude ${args.join(' ')}`;

    console.log(`[agent:repl] Starting REPL: container=${containerId.slice(0, 12)} agent=${config.agentName}`);

    SandboxManager.execStream(containerId, ['sh', '-c', shellCmd], {
      workDir,
      keepStdinOpen: true,
      onStdin: (stdin) => {
        this.stdinStream = stdin;
      },
      onStdout: (chunk) => {
        if (this.killed) return;
        this.partialLine += chunk;
        const lines = this.partialLine.split('\n');
        this.partialLine = lines.pop() ?? '';
        for (const line of lines) {
          if (!line.trim()) continue;
          const event = EventParser.parseLine(line);
          if (event) {
            const unified = EventParser.toUnified(event);
            if (unified) this.emit(unified);
          }
        }
      },
      onStderr: (chunk) => {
        if (this.killed) return;
        const msg = chunk.trim();
        if (msg) this.emit({ type: 'error', message: msg, providerRaw: msg, timestamp: Date.now() });
      },
    }).catch((err) => {
      if (!this.killed) {
        this.emit({ type: 'error', message: `Docker exec error: ${err.message}`, timestamp: Date.now() });
      }
    });
  }

  sendPrompt(prompt: string): void {
    if (this.stdinStream && !this.killed) {
      try {
        // Claude Code REPL reads one line at a time.
        // Multi-line prompts need careful handling — escape newlines.
        const escaped = prompt.replace(/\n/g, '\\n').replace(/'/g, "'\\''");
        this.stdinStream.write(`echo '${escaped}' | xargs -0 printf '%b' && echo '' ; true\n`);
        // Simpler: write to a temp file and cat it
        // For now, single-line prompts:
        this.stdinStream.write(prompt.replace(/\n/g, ' ') + '\n');
      } catch { /* process already exited */ }
    }
  }

  write(input: string): void {
    if (this.stdinStream && !this.killed) {
      try { this.stdinStream.write(input); } catch { /* ignore */ }
    }
  }

  stop(): void {
    this.killed = true;
    if (this.stdinStream) {
      try { this.stdinStream.end(); } catch { /* ignore */ }
      this.stdinStream = null;
    }
    if (this.containerId) {
      SandboxManager.execShell(this.containerId, 'pkill -f "claude.*--verbose" 2>/dev/null || true');
    }
  }
}
```

// apps/api/src/agent/EventParser.ts 新增方法
static toUnified(event: ParsedEvent): UnifiedAgentEvent | null {
  const base = { providerRaw: event, timestamp: Date.now() };
  switch (event.type) {
    case 'text':                return { ...base, type: 'thinking', content: event.content };
    case 'tool_use':            return { ...base, type: 'tool_use', toolName: event.toolName, toolInput: event.input };
    case 'tool_result':         return { ...base, type: 'tool_result', content: event.content };
    case 'subagent_start':      return { ...base, type: 'subagent_start', content: event.agentType };
    case 'subagent_result':     return { ...base, type: 'subagent_result', content: event.agentType };
    case 'permission_request':  return { ...base, type: 'permission_request', toolName: event.tool, filePath: event.path };
    case 'done':                return { ...base, type: 'done', exitCode: event.exitCode };
    case 'error':               return { ...base, type: 'error', message: event.message };
    default:                    return null;
  }
}

// apps/api/src/agent/providers/factory.ts
import { AbstractProvider } from './base.js';
import { ClaudeCodeProvider } from './claude-code.js';

const registry = new Map<string, () => AbstractProvider>();

export class ProviderFactory {
  static register(name: string, factory: () => AbstractProvider): void {
    registry.set(name, factory);
  }

  static create(providerName: string): AbstractProvider {
    const factory = registry.get(providerName);
    if (!factory) {
      throw new Error(
        `Unknown provider: ${providerName}. Available: ${[...registry.keys()].join(', ')}`
      );
    }
    return factory();
  }

  static list(): string[] {
    return [...registry.keys()];
  }

  /** Initialize built-in providers on startup */
  static init(): void {
    ProviderFactory.register('claude-code', () => new ClaudeCodeProvider());
  }
}

## Task 0-8: Fix Docker Stdin Multiplex Issue

**Files:**
- Modify: `apps/api/src/agent/SandboxManager.ts`

**Problem**: When `keepStdinOpen: true` and no initial stdin data is written, Docker's multiplexed stdout demux can stall (observed in Phase 2 testing). The `cat file - | claude` pattern requires persistent stdin.

**Fix**: Write a priming newline to stdin immediately after exec starts:

```typescript
if (opts.keepStdinOpen && streamAny.stdin) {
  // Prime the stdin stream — Docker needs at least one write to properly
  // initialize the multiplexed stdin channel. A newline is harmless; cat - reads
  // it and passes it to claude which treats it as a blank input separator.
  streamAny.stdin.write('\n');
  if (opts.onStdin) opts.onStdin(streamAny.stdin);
}
```
